1. 필수 포함 정보

■ Context (맥락)
  
  ● 데이터셋이 무엇에 관한 것인가? 어떻게 사용되었는가?
  
  ● 분석을 통해 달성하고자 하는 목표가 무엇인가?
  
    ○ 리포트가 단순한 결과 나열이 되지 않도록 주의
      예: "이 알고리즘을 적용했더니 이런 결과가 나왔고, 
           그 다음 저 알고리즘을 적용했더니 저런 결과가 나왔다" (X)
    
    ○ 알고리즘 적용 전에 "무엇을 하려고 하는지" 설명 필수
  
  ● 데이터셋의 특성 파악 및 이해
  
    ○ 데이터 탐색(Data Exploration)을 통해
    
    ○ 단, 발견한 모든 것을 보고하지 말 것
      → 유용하거나 흥미로운 것만 선별해서 보고
    
    ○ 컬럼과 값의 의미를 설명

■ 데이터 과학 프로토콜 명확히 설명
  
  ● 어떤 전처리를 했는가? 왜 했는가?
  
  ● Training/Test 데이터를 어떻게 나눴는가?
    각각에 어떤 작업을 수행했는가?

■ 결과 분석
  
  ● 단순히 "여기 결과가 있습니다"만 보여주지 말 것
  
    ○ 예상한 결과인가? 아니면 놀라운 결과인가?
    
    ○ 왜 이런 결과가 나왔는지 설명을 시도
    
    ○ 하이퍼파라미터 수정이 왜 더 나은 결과를 가져왔는지 등
  
  ● 결과들을 연결하기
  
    ○ 분석 경로(Analysis Path)를 포함하면 리포트가 더 흥미로워짐
    
    ○ 예시: "처음에 특정 방법을 적용했는데, 
             특정 현상을 관찰했고, 
             이것이 추가 전처리나 하이퍼파라미터 변경의 아이디어를 주었다"

3. 중요한 정보만 추가하기
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

■ 너무 많이 쓰지 말 것

■ 정보를 정제한 후 리포트에 포함

  ● Python 노트북의 출력을 그대로 복사-붙여넣기 하지 말 것
    → 정보가 있는 부분만 선별
  
  ● 카테고리가 정수로 인코딩되어 있다면
    → 숫자 대신 카테고리 이름을 표시하는 것이 좋음
    예: 0, 1, 2 (X) → Low, Medium, High (O)
  
  ● 필요하지 않다면 너무 많은 소수점을 사용하지 말 것
    예: 5.7598766% (X) → 5.8% (O)
    
    명확성을 위해 색상 사용 가능
    예: 녹색(좋음), 노란색(보통), 주황색(주의), 빨간색(나쁨)
  
  ● 추가하는 그림이나 그래프가 읽기 쉬운지 확인
    → 폰트 크기가 충분히 큰지
    → 해상도가 좋은지

8. 예시: 좋은 vs 나쁜 표현
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

■ 나쁜 예 ❌
  "Logistic Regression을 적용했더니 accuracy가 0.67982343이 나왔다.
   그 다음 Random Forest를 적용했더니 0.75123456이 나왔다."

■ 좋은 예 ✅
  "데이터의 선형 관계를 먼저 파악하기 위해 Logistic Regression을 
   baseline으로 선택했다 (Accuracy: 68%). 
   
   결과가 예상보다 낮았는데, EDA에서 발견한 비선형 패턴 때문으로 보인다.
   
   이를 해결하기 위해 Random Forest를 시도했고 (Accuracy: 75%), 
   성능이 개선되었다. 
   
   그러나 feature importance 분석 결과 일부 변수들이 overfitting을 
   유발할 가능성이 있어 추가 검증이 필요하다."
